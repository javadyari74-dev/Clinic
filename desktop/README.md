# نرم‌افزار دسکتاپ مطب زیبایی دکتر یاری (Windows)

این پوشه کل پروژه را به یک **نرم‌افزار مستقل ویندوز** تبدیل می‌کند: سرور، رابط
کاربری و دیتابیس SQLite همگی داخل یک فایل نصب‌کننده `.exe` بسته‌بندی می‌شوند.
کاربر نهایی به هیچ نرم‌افزار دیگری (Node.js، مرورگر و …) نیاز ندارد.

از نظر فنی از **Electron** (که موتور Node و مرورگر داخلی را با خود حمل می‌کند) و
**electron-builder** برای ساخت فایل نصب‌کننده NSIS استفاده می‌شود.

---

## مهم: چرا ساخت روی ویندوز با کل پروژه شکست می‌خورد

محیط اصلی پروژه (`pnpm-workspace.yaml`) عمداً فقط برای لینوکس تنظیم شده و باینری
ویندوزِ ابزارهای ساخت (esbuild و lightningcss) را حذف می‌کند. به همین دلیل اگر
روی ویندوز دستور `pnpm build` اجرا شود، با خطا (`ERR_PNPM_IGNORED_BUILDS`) متوقف
می‌شود.

**راه‌حل ساده:** بخش‌های جاوااسکریپتیِ برنامه (رابط کاربری و سرور) فایل‌های مستقل
از سیستم‌عامل‌اند و یک‌بار روی لینوکس (Replit) ساخته شده و داخل پوشهٔ
`desktop/dist` قرار گرفته‌اند. پس روی ویندوز **نیازی به اجرای pnpm یا ساختِ دوباره
نیست**؛ اسکریپت `assemble` به‌طور خودکار از همان بسته‌های آماده استفاده می‌کند و
فقط `electron-builder` (بسته‌بندی نهایی) و باینری ویندوزِ دیتابیس اجرا می‌شوند.

> اگر `desktop/dist` از قبل ساخته شده باشد (که در بستهٔ آماده همین‌طور است)،
> مرحلهٔ ساختِ workspace به‌طور کامل رد می‌شود.

## پیش‌نیازها (روی کامپیوتر ویندوز)

1. **Node.js نسخه ۱۸ یا بالاتر** — از https://nodejs.org نصب کنید.

> دیگر نیازی به نصب pnpm یا ساختِ کل پروژه روی ویندوز نیست.

## مراحل ساخت فایل نصب (با بستهٔ آماده)

پوشهٔ `desktop` (که شامل `dist` آمادهٔ ساخته‌شده است) را روی ویندوز قرار دهید،
سپس داخل همان پوشه:

```powershell
cd desktop
npm install
npm run dist
```

> در پوشه `desktop` از **npm** استفاده کنید (نه pnpm)، چون این پوشه عمداً از فضای
> کاری pnpm جداست. `npm install` فقط Electron و باینری ویندوزِ دیتابیس را نصب
> می‌کند.

پس از پایان، فایل نصب‌کننده در این مسیر ساخته می‌شود:

```
desktop/release/DoctorYariClinic-Setup-1.0.0.exe
```

این فایل را اجرا کنید تا نرم‌افزار نصب شود و آیکون روی دسکتاپ ساخته شود.

## اجرای آزمایشی بدون ساخت نصب‌کننده

برای تست سریع برنامه به‌صورت دسکتاپ (بدون ساختن `.exe`):

```powershell
cd desktop
npm install
npm start
```

## ساخت فقط پوشه برنامه (بدون نصب‌کننده)

```powershell
npm run dist:dir
# خروجی در desktop/release/win-unpacked
```

## نحوه کارکرد

- هنگام اجرا، نرم‌افزار یک سرور داخلی (Express) را روی پورت `47615` بالا می‌آورد
  (اگر اشغال بود، پورت آزاد بعدی). همان سرور هم رابط کاربری و هم `/api` را سرو
  می‌کند، پس همه‌چیز از یک آدرس واحد بالا می‌آید.
- یک پنجره برنامه باز می‌شود و مستقیماً پنل مدیریت مطب را نشان می‌دهد.
- در اولین اجرا، کاربر مدیرکل به‌صورت خودکار ساخته می‌شود:
  - نام کاربری: `admin`
  - رمز عبور: `admin123`

## محل ذخیره داده‌ها

دیتابیس در پوشه داده‌های کاربر ویندوز ذخیره می‌شود (نه داخل پوشه نصب)، تا با
به‌روزرسانی یا نصب مجدد نرم‌افزار، داده‌ها پاک نشوند:

```
C:\Users\<نام کاربر>\AppData\Roaming\doctor-yari-clinic-desktop\clinic.db
```

برای پشتیبان‌گیری، علاوه بر همین فایل، از بخش «پشتیبان‌گیری» داخل خود نرم‌افزار
هم می‌توانید فایل بک‌آپ بگیرید.

## دسترسی از گوشی روی همان وای‌فای

سرور روی همه آدرس‌های شبکه گوش می‌دهد. از منوی **برنامه → نمایش آدرس دسترسی از
گوشی**، آدرس IP کامپیوتر و پورت را ببینید (مثل `http://192.168.1.50:47615`) و آن
را در مرورگر گوشی (متصل به همان وای‌فای) باز کنید. ورود با همان کاربر و رمز است.

> اگر فایروال ویندوز اجازه نداد، هنگام اولین اجرا روی «Allow access» کلیک کنید.

## آیکون اختصاصی (اختیاری)

برای آیکون اختصاصی، یک فایل `build/icon.ico` (حداقل ۲۵۶×۲۵۶) در پوشه `desktop`
قرار دهید؛ electron-builder به‌صورت خودکار از آن استفاده می‌کند. در نبود آن،
آیکون پیش‌فرض Electron استفاده می‌شود.

## نکته درباره اخطار ویندوز

چون نرم‌افزار امضای دیجیتال (code signing) ندارد، ممکن است ویندوز هنگام نصب پیام
«Windows protected your PC» نشان دهد. روی **More info → Run anyway** بزنید.
